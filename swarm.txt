Step 1: Identify the Host IP (System A)
First, you need the IPv4 address of System A on your local network. Both systems must be connected to the same Wi-Fi or Ethernet network.

On System A, open the Command Prompt (Windows) or Terminal (Mac/Linux).

Type ipconfig (Windows) or ifconfig (Mac/Linux).

Locate your IPv4 Address (it will usually look something like 192.168.1.100 or 10.0.0.5).
Write this IP address down. We will refer to it as <SYSTEM_A_IP>.

Step 2: Unbind the Ganache Blockchain (System A)
By default, Ganache only accepts transactions from its own machine. We must expose it to the LAN.

Open the Ganache UI on System A.

Click the Gear Icon (Settings) in the top right.

Go to the Server tab.

Under the Hostname dropdown, change it from 127.0.0.1 - lo to either your <SYSTEM_A_IP> or 0.0.0.0 - All Interfaces.

Click Save and Restart. Ganache is now broadcasting the ledger to your local network on port 7545.

Step 3: Unbind the IPFS Daemon (System A)
The IPFS Kubo node must also allow System B to fetch the decrypted payloads.

Open a terminal on System A.

Tell the IPFS API to listen to the network:
ipfs config Addresses.API /ip4/0.0.0.0/tcp/5001

Tell the IPFS HTTP Gateway to serve files to the network:
ipfs config Addresses.Gateway /ip4/0.0.0.0/tcp/8080

Restart the IPFS daemon:
ipfs daemon

Step 4: Update the DApp Frontend Code (System A)
The DApp HTML file currently hardcodes 127.0.0.1. If System B loads the DApp, it will try to look for IPFS on its own machine, which will fail.

Open your index.html file in your code editor on System A.

Locate the IPFS_GATEWAY constant (around line 188).

Change it from the localhost address to System A's LAN IP:

JavaScript
// Change this:
// const IPFS_GATEWAY = "http://127.0.0.1:8080/ipfs/";

// To this (replace with your actual IP):
const IPFS_GATEWAY = "http://<SYSTEM_A_IP>:8080/ipfs/";
Save the file.

Boot your simple HTTP server on System A so System B can access the webpage:
python -m http.server 8000 --bind 0.0.0.0

Step 5: Configure the Firewall (System A)
Windows Defender or local firewalls will likely block inbound requests to these new ports.

On System A, open your Firewall Settings (e.g., "Windows Defender Firewall with Advanced Security").

Create Inbound Rules to allow TCP traffic on ports 7545 (Ganache), 5001 (IPFS API), 8080 (IPFS Gateway), and 8000 (Python Web Server).

Step 6: Connect the Client (System B)
Now, move over to the second physical computer (System B).

Configure MetaMask: Open the MetaMask extension. Add a new network exactly as you did before, but for the RPC URL, enter: http://<SYSTEM_A_IP>:7545.

Import a Wallet: Import one of the private keys generated by Ganache (e.g., Account Index 1 or 2) to act as your Scientist's wallet.

Access the DApp: Open the web browser on System B and navigate to: http://<SYSTEM_A_IP>:8000.
